# Slide Structure Designer Skill

주어진 입력을 바탕으로
**강의, 세일즈, 컨설팅 목적의 발표에 적합한**
발표 흐름이 명확한 PPT 슬라이드 구조를 설계하는 스킬입니다.

---

## 언제 사용되는가

다음과 같은 상황에서 이 스킬을 사용합니다.

- PPT, 발표자료, 프레젠테이션 구조 설계 요청 시
- 결과물이 **한국어 콘텐츠**일 때
- **강의, 세일즈, 컨설팅 목적의 발표**일 때
- 정보 나열이 아닌 **메시지 중심의 전달과 전환**이 필요한 경우

---

## 필수 입력 정보

이 스킬은 다음 4가지 정보가 필요합니다.

| 항목 | 설명 | 예시 |
|------|------|------|
| **주제** | 발표의 핵심 주제 | AI 도구를 활용한 업무 자동화 |
| **대상** | 발표를 듣는 청중 | 중소기업 마케팅 담당자 |
| **목표** | 발표 후 유도할 행동 | AI 도구 도입 상담 신청 |
| **슬라이드 수** | 전체 슬라이드 개수 | 12장 |

---

## 작동 순서

### Step 1: 정보 수집 (AskUserQuestion 활용)

사용자가 필요한 정보를 모두 제공하지 않은 경우, **AskUserQuestion 툴**을 사용하여 누락된 정보를 수집합니다.

**수집 방식:**

1. **발표 유형 확인** (정보 누락 시)
   ```
   question: "어떤 유형의 발표인가요?"
   header: "발표 유형"
   options:
     - label: "강의/교육"
       description: "청중에게 지식이나 스킬을 전달하는 발표"
     - label: "세일즈/제안"
       description: "제품이나 서비스 구매를 유도하는 발표"
     - label: "컨설팅/보고"
       description: "분석 결과나 전략을 제안하는 발표"
   ```

2. **슬라이드 수 확인** (정보 누락 시)
   ```
   question: "슬라이드는 몇 장이 필요한가요?"
   header: "슬라이드 수"
   options:
     - label: "8~10장"
       description: "핵심만 전달하는 간결한 발표 (10-15분)"
     - label: "12~15장"
       description: "충분한 설명이 포함된 표준 발표 (20-30분)"
     - label: "18~20장"
       description: "상세한 내용을 다루는 심화 발표 (40분 이상)"
   ```

3. **주제, 대상, 목표**는 자유 입력이 필요하므로 텍스트로 질문합니다.
   - 사용자가 "Other"를 선택하면 직접 입력할 수 있습니다.

### Step 2: 구조 설계

수집된 정보를 바탕으로 슬라이드 구조를 설계합니다.

- 강의, 세일즈, 컨설팅 상황에 맞게 발표 흐름을 자연스럽게 구성
- 전체 구조는 다음 비율을 기준으로 배분:
  - **초반 30%**: 문제 인식 및 공감
  - **중반 40%**: 구조 제안 및 관점 정리
  - **후반 30%**: 제안 및 전환 유도

### Step 3: 결과 검토 요청 (선택)

구조 설계 완료 후, 사용자에게 검토 의견을 받을 수 있습니다.

```
question: "설계된 구조를 검토해주세요. 수정이 필요한 부분이 있나요?"
header: "구조 검토"
options:
  - label: "좋습니다"
    description: "현재 구조로 진행합니다"
  - label: "초반 강화"
    description: "문제 인식/공감 부분을 더 강조해주세요"
  - label: "후반 강화"
    description: "제안/전환 유도 부분을 더 강조해주세요"
```

---

## 설계 규칙

1. 슬라이드는 요청된 개수로만 구성합니다.
2. 각 슬라이드는 **하나의 핵심 메시지**만 가집니다.
3. 제목은 설명이 아닌 **메시지를 전달하는 단정형 문장**으로 작성합니다.
4. 추상적인 표현은 사용하지 않습니다.
5. 한국어 가독성과 발표 전달력을 고려해 문장은 짧고 명확하게 작성합니다.

---

## 출력 형식

각 슬라이드는 아래 형식으로 출력합니다.

```
[슬라이드 번호] - [구간: 문제인식/구조제안/전환유도]
제목: (단정형 메시지)
핵심 메시지: (청중이 기억해야 할 한 문장)
```

**출력 예시:**

```
[1] - 문제인식
제목: 마케팅 담당자의 80%가 반복 업무에 지쳐있다
핵심 메시지: 매일 같은 작업을 반복하며 시간을 낭비하고 있습니다

[2] - 문제인식
제목: 수작업 리포트 작성에 하루 3시간을 쓰고 있다
핵심 메시지: 이 시간을 전략적 업무에 쓸 수 있다면?

[3] - 구조제안
제목: AI 자동화로 반복 업무를 90% 줄일 수 있다
핵심 메시지: 검증된 AI 도구들이 이미 존재합니다
```

---

## 검증 기준

결과물은 다음 기준을 만족해야 합니다.

- [ ] 제목만 읽어도 전체 흐름이 이해되는가
- [ ] 슬라이드만 훑어봐도 메시지가 전달되는가
- [ ] 강의, 세일즈, 컨설팅 목적에 맞는 전환 구조인가
- [ ] 초반 30%에서 문제 인식이 충분히 이루어졌는가
- [ ] 각 제목이 단정형 문장으로 작성되었는가
