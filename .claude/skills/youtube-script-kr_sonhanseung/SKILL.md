# 한국 유튜브 대본 제작 스킬

내향적 커뮤니케이션 채널을 위한 주제 발굴 → 대본 작성 → 패키징.

---

## STEP 0: 콘텐츠 방향 설정 (AskUserQuestion 필수)

**워크플로우 시작 전, 반드시 AskUserQuestion으로 핵심 설정 수집**

### 빠른 시작 옵션

사용자가 "빠르게", "기본값으로", "바로 시작" 등을 요청하면:
- 주제 카테고리: 관계의 법칙
- 영상 스펙: 10분 × 3개
- 톤 강도: 중간
- 주제 선택: Claude 자동 선택

으로 바로 진행.

## 전체 제작 워크플로우

사용자가 "유튜브 영상 N개 만들어줘" 또는 "대본 패키지 만들어줘" 요청 시 실행.

### STEP 1: 날카로운 주제 발굴 (자동)

**목표:** 한국 유튜브에서 아무도 안 다루는 날카로운 주제 발굴

**실행 방법:**
1. 4가지 발견 방법론 모두 적용
   - 모순 찾기
   - 반복되는 고통 지점
   - 작은 불편함
   - 설명되지 않는 관찰

2. 주제 좁히기 프로세스 적용
   ```
   넓은 주제 → 상황 구체화 → 순간 구체화 → 모순 찾기 → 최종 날카로운 주제
   ```

3. 생성 개수: 사용자 요청의 **2배** 생성
   - 요청 5개 → 10개 주제 생성
   - 요청 10개 → 20개 주제 생성

4. 각 주제 포맷:
   ```
   제목: "3번 거절하면 사라지는 사람이 진짜 친구다"
   통념: 친구는 끝까지 포기 안 함
   모순: 진짜 친구는 경계 존중해서 물러남
   왜 날카로운가: 우정 = 집착 공식 깨뜨림
   ```

**출력 형식:**
```
📋 날카로운 주제 [N]개 발굴 완료

【블록 1: 관계의 잔인한 법칙】
1. "3번 거절하면 사라지는 사람이 진짜 친구다"
   통념 vs 모순 + 왜 날카로운가

2. "왜 고민 상담한 사람과는 멀어지는가"
   통념 vs 모순 + 왜 날카로운가

【블록 2: 조직 생존의 암묵적 규칙】
...
```

### STEP 2: 주제 선택

**STEP 0에서 선택한 방식에 따라 진행:**

#### 옵션 A: Claude 자동 선택 (STEP 0에서 선택 시)
- 기준: 통념과 모순의 격차가 큰 것
- 기준: 한국 문화 특수성 반영
- 기준: 실생활 적용 가능성
- 선택 후 간략히 선택 이유 설명

#### 옵션 B: 사용자 직접 선택 (STEP 0에서 선택 시)

**AskUserQuestion으로 주제 선택 받기:**
```
question: "발굴된 주제 중 어떤 것으로 진행할까요?"
header: "주제 선택"
options:
  - label: "[1번 주제 제목]"
    description: "통념 vs 모순 요약"
  - label: "[2번 주제 제목]"
    description: "통념 vs 모순 요약"
  - label: "[3번 주제 제목]"
    description: "통념 vs 모순 요약"
  - label: "[4번 주제 제목]"
    description: "통념 vs 모순 요약"
multiSelect: true  # 여러 개 선택 가능
```

필요한 개수만큼 여러 라운드로 질문 (한 번에 4개씩)

#### 옵션 C: 특정 상황 있음 (STEP 0에서 선택 시)

**AskUserQuestion으로 상황 파악:**
```
question: "어떤 상황이나 경험을 다루고 싶으세요?"
header: "상황 설명"
options:
  - label: "직장에서 겪은 일"
    description: "회의, 상사, 동료 관계 등"
  - label: "인간관계에서 겪은 일"
    description: "친구, 연인, 가족과의 상황"
  - label: "혼자 느끼는 감정"
    description: "외로움, 불안, 자기 의심 등"
  - label: "직접 설명할게요"
    description: "구체적인 상황을 텍스트로 입력"
multiSelect: false
```

→ 사용자 입력을 바탕으로 맞춤형 주제 발굴 후 다시 선택

### STEP 3: 웹 리서치 (자동)

**선택된 각 주제마다:**

1. 관련 심리학/신경과학 연구 검색
2. 권위 학자 1-2명 선정
3. 핵심 개념 추출

**리서치 예시:**
```
주제: "왜 고민 상담한 사람과는 멀어지는가"
→ 검색: "vulnerability disclosure psychology"
→ 선정: 브레네 브라운 (취약성 연구)
→ 핵심: 취약성 노출은 양날의 검
```

### STEP 4: 대본 작성 (자동)

**각 주제마다 6단계 소제목 구조 대본 작성:**

```
■ [1단계 제목] 개인 경험 → 자기 의심
(300-400자) ← 2단 구조로 확장

■ [2단계 제목] 통념 → 사회 학습 구조
(300-400자) ← 왜 그렇게 믿게 됐는지 설명

■ [3단계 제목] 오해 차단 → 핵심 주장
(200-250자) ← 논점 명확화 후 주장

■ [4단계 제목] 구조 해체 (외적 행동 + 내적 심리)
(600-800자) ← 가장 긴 핵심 구간
- 권위 인용 포함
- 5-7단계 메커니즘 (각 단계마다 A의 생각 추가)
- 역할 고착 설명 추가

■ [5단계 제목] 판단 기준 제시
(400-500자) ← 행동 지침이 아닌 판단 질문
- 3가지 판단 질문
- 각 질문의 작동 원리 설명

■ [6단계 제목] 불편한 진실 + 질문
(100-150자)
```

**총 분량:** 1,900-2,500자 (10-12분 영상)
**핵심:** 4단계가 가장 길고, 1-2-5단계가 중간, 3-6단계가 짧음

### STEP 5: 대본 검토 및 피드백

**첫 번째 대본 완성 후 AskUserQuestion으로 피드백 수집:**
```
question: "첫 번째 대본입니다. 톤과 방향이 맞나요?"
header: "대본 피드백"
options:
  - label: "좋아요, 이대로 진행"
    description: "나머지 대본도 같은 톤으로"
  - label: "더 날카롭게"
    description: "통념과 모순의 대비를 강화"
  - label: "조금 부드럽게"
    description: "공감대 형성을 더 강조"
  - label: "구조 수정 필요"
    description: "특정 부분 수정 요청"
multiSelect: false
```

**피드백 반영 후 나머지 대본 작성**

### STEP 6: 비롤 + 설명 추가

**각 대본마다 자동 추가:**

1. **비롤 아이디어 6개**
   - 구체적으로 촬영 가능한 장면
   - 예: "스마트폰 화면 - 카톡 목록, 마지막 대화 날짜 3개월 전"

2. **스토리텔링 설명 (5-7줄)**
   - 핵심 문장 1개
   - 구조 설명
   - 긴장-해결 포인트
   - 마지막 질문의 의도

### STEP 7: 최종 패키징

**단일 DOCX 파일 생성:**

```
📄 유튜브_영상_완전_패키지_[N]편.docx

├─ 표지
├─ 목차
├─ 영상 1
│  ├─ 대본 (소제목 구조)
│  ├─ 비롤 아이디어
│  └─ 스토리텔링 설명
├─ 영상 2-N (동일 구조)
└─ 제작 가이드
```

**파일 위치:** `/mnt/user-data/outputs/`

## 워크플로우 단축 명령

사용자가 이렇게 요청하면 워크플로우 실행:

- "유튜브 영상 5개 만들어줘"
- "대본 패키지 10개"
- "날카로운 주제로 영상 3개"
- "내향형 콘텐츠 패키지 만들어줘"

**실행 순서:**
1. **STEP 0: AskUserQuestion으로 방향 설정** (4개 질문)
2. **STEP 1:** 주제 발굴 (요청 개수 x2)
3. **STEP 2:** 주제 선택 (자동 or AskUserQuestion)
4. **STEP 3:** 웹 리서치
5. **STEP 4:** 대본 작성
6. **STEP 5:** 대본 피드백 (AskUserQuestion)
7. **STEP 6:** 비롤 + 설명 추가
8. **STEP 7:** DOCX 패키징
9. 파일 제시

**빠른 진행 모드:** "빠르게 만들어줘" 요청 시 기본값으로 진행, 피드백 단계만 유지.

## 핵심 원칙

### 채널 정체성

**장르:** 사고 해설형 인문 커뮤니케이션 콘텐츠  
**NOT:** 자기계발, 힐링, 화술 강의, 동기부여

**타겟:**
- 20대 후반~40대 초반 직장인/프리랜서/1인 사업자
- 내향적 성향으로 자기 정의
- 말 잘하는 사람 되기보다 신뢰/존중/깊이 추구
- 조용하면 진다 서사에 피로감
- 빠른 스킬/공식/템플릿과 감정적 위로 회피

**톤:**
- 차분하고 절제된 해설자 시점
- 최소 감정, 급한 결론 회피, 여운 남기기
- 금지 - 과도한 공감, 감정 조작, 자기 고백, 조언 남발

### 대본 10대 원칙

1. **절대 가르치지 않고, 해석만 한다** - 화자는 분석가/해설자
2. **나 이야기 최소화** - 관찰 대상이 주인공
3. **고정 구조** - 통념 → 균열 → 구조적 설명
4. **감정 조작 금지** - 목표는 인지적 이해
5. **생각 정렬에 집중** - 사고 속도, 발화 밀도 분석
6. **짧은 결론** - 한 문장 요약 또는 질문
7. **과잉 설명 회피** - 해석 공백 남기기
8. **느린 템포** - 정보 밀도 낮춤
9. **판단 회피** - 옳고 그름 없음
10. **자기 포지셔닝 금지** - 신뢰는 콘텐츠로

## 6단계 소제목 구조 (상세)

### 1단계: 개인 경험 → 자기 의심 (2단 구조, +60~80초)
**1-1. 경험 고백**
- 숫자로 시작 (몇 명, 몇 번, 몇 년)
- 구체적 상황 묘사
- 예: "저는 정말 친한 사람이 세 명 있습니다"

**1-2. 자기 의심 (필수 추가)**
사고 과정을 공개합니다. 감정 토로가 아닙니다.
- "이게 성숙한 건가, 아니면 습관인가?"
- "관계를 지킨 건지, 피한 건지?"
- "내가 착해서 그런 걸까, 아니면 불편함을 못 견뎌서일까?"
→ 청자가 "아, 이 사람도 헷갈렸구나" 하고 따라옵니다.

### 2단계: 통념 → 사회 학습 구조 (+70~90초)
**단순 상식 나열이 아니라, 왜 그렇게 믿게 됐는지 설명**

구조:
- 가정: "참는 사람이 어른"
- 학교: "문제 일으키지 않는 애가 착한 애"
- 조직: "먼저 숙이는 사람이 팀워크 좋음"
- 관계: "먼저 연락하는 쪽이 더 좋아하는 사람"

핵심 문장 (필수):
- "우리는 사과를 '선택'이 아니라 '역할'로 배워왔습니다."

→ 청자의 과거를 해석해주는 구간. 체류 시간이 깁니다.

### 3단계: 오해 차단 → 핵심 주장 (2단 구조, +40~50초)
**3-1. 오해 차단 (필수)**
논점을 명확히 좁혀야 반발이 줄어듭니다.
- "오늘 얘기는 '사과하면 안 된다'가 아닙니다."
- "문제는 '항상 먼저'입니다."
- "한쪽만 계속 먼저라는 구조입니다."

**3-2. 핵심 주장**
- "그런데 실제로는"
- 통념과 현실의 괴리
- 충격적 반전

### 4단계: 구조 해체 - 외적 행동 + 내적 심리 (+2~3분)
**가장 크게 늘려야 하는 핵심 구간**

권위 학자 인용 (이름만, 수식어 없이)

**5-7단계 메커니즘: 각 단계마다 A의 생각 추가 (필수)**
```
1단계: [외적 행동] + A의 생각: "일단 끝내자"
2단계: [상대 해석] + A의 생각: "이번에도 내가 정리하면 되겠지"
3단계: [관계 변화] + A의 생각: "왜 나만 이걸 해야 하지?"
4단계: [고착화] + A의 생각: "이걸 안 하면 내가 나쁜 사람이 되네"
5단계: [최종 결과] + A의 생각: "관계를 지키는 게 아니라, 나를 잃고 있구나"
```
→ 단계 하나당 30~40초. 5단계면 2분 이상 확보.

**역할 고착 설명 추가 (+60~80초, 필수)**
- 문제를 만든 사람 ≠ 문제를 해결하는 사람
- 그런데 항상 해결하는 사람이 고정되면
- 관계는 원인 추적을 멈춥니다
- "누가 잘못했는지"가 아니라
- "누가 처리하느냐"만 남습니다
→ 지적 만족도가 높아서 청자가 집중합니다.

### 5단계: 판단 기준 제시 (+1~1.5분)
**행동 지침이 아니라 판단 질문으로**

각 대안을 질문 형태로:
```
첫 번째 판단: "이 사과는 문제 해결인가, 역할 수행인가?"
[질문의 작동 원리 설명]

두 번째 판단: "기다렸을 때 불안해진다면, 그 불안의 주인은 누구인가?"
[질문의 작동 원리 설명]

세 번째 판단: "이 갈등이 정말 사과가 필요한가, 불편함을 견디면 끝나는가?"
[질문의 작동 원리 설명]
```
→ 조언이 아니라 사고 도구가 됩니다. 분량도 자연스럽게 늘어납니다.

### 6단계: 불편한 진실 + 질문
- 도덕적 위로 없음
- 현실 그대로 제시
- 마지막 질문으로 자기 점검
- 예: "당신은 지금 [A]하고 있는가, 아니면 [B]하고 있는가"

## 문장 특성: 자연스럽게 말하는 원칙

### ⚠️ 절대 금지: 설명용 문장
대본은 **말하는 문장**이어야 합니다. 논문·보고서·강의 원고가 아닙니다.

**❌ 설명용 문장 (절대 사용 금지):**
- "~입니다." 연속 3번 이상
- "~합니다." "~됩니다." 반복
- "~라고 불렀습니다." (학술 인용체)
- "메커니즘은 다음과 같이 작동합니다." (보고서체)
- "이유는 다음과 같습니다." (발표문체)

### 6대 자연스러운 문장 원칙

#### 원칙 1: 문장의 30-40%는 미완 문장으로
```
❌ 먼저 사과하는 사람이 관계에서 집니다.
⭕ 이상하게요. 먼저 사과하는 쪽이, 결국 더 힘들어집니다.

❌ 상대는 사과를 기다리게 됩니다.
⭕ 상대는요, 기다리게 됩니다. 사과를.
```

**적용 방법:**
- "~요", "~고요", "~는데요"로 문장 중간 끊기
- 주어와 술어 사이에 쉼표
- 도치 구조 활용 (술어 → 목적어)

#### 원칙 2: 설명 앞에 '말 걸기 문장' 붙이기
```
❌ 우리는 이렇게 생각합니다. 먼저 사과하는 사람이 성숙하다.
⭕ 보통은 이렇게 생각하죠. 먼저 사과하는 사람이 어른이라고.

❌ 메커니즘은 다섯 단계로 작동합니다.
⭕ 어떻게 이런 일이 벌어지냐면요. 크게 다섯 단계입니다.
```

**말 걸기 문장 예시:**
- "보통은 ~하죠"
- "이상하게도요"
- "생각해보면"
- "실제로는요"
- "그런데 말입니다"

#### 원칙 3: 문단마다 '숨 쉬는 문장' 삽입
논리적으로 불필요해도, 사람이 말하고 있다는 신호가 됩니다.

**숨 쉬는 문장 예시:**
- "이쯤 되면 좀 이상하죠."
- "여기서부터 관계가 달라집니다."
- "이걸 겪고 나서야 알았습니다."
- "근데 진짜 문제는 여기서부터입니다."
- "조금 불편하지만 계속 들어보세요."

#### 원칙 4: 결론형 : 과정형 = 3 : 7
모든 문장이 "~입니다"로 끝나면 리듬이 죽습니다.

```
❌ 결론형 연속
먼저 사과하는 사람이 관계에서 집니다. 
계속 먼저 사과하면 상대는 기다리게 됩니다.
문제를 해결하는 쪽이 책임자가 됩니다.

⭕ 과정형 혼합
이상하죠. 먼저 사과하면, 오히려 일이 더 커집니다. 
상대는 기다리게 되고요. 나중엔 요구하게 됩니다.
그러니까 결국, 책임이 이쪽으로 넘어오는 겁니다.
```

**과정형 문장 패턴:**
- "~하면, ~됩니다"
- "~하게 되고요"
- "~는 겁니다"
- "~인 거죠"
- "~게 만듭니다"

#### 원칙 5: 접속사와 숨 고르기 단어 활용
실제 말에는 접속사가 많습니다. 생각을 정리하면서 말하기 때문입니다.

**필수 접속사:**
- "그러니까"
- "근데"
- "사실"
- "그래서"
- "그런데 말입니다"
- "다시 말하면"

**사용 예:**
```
❌ 접속사 없음
A가 먼저 사과합니다. B는 기다리게 됩니다.

⭕ 접속사 있음
A가 먼저 사과합니다. 그러니까 B는요, 기다리게 됩니다.
```

#### 원칙 6: 마지막은 '정리'이지 '판정'이 아님
딱 떨어지는 결론은 강의입니다. 여운을 남겨야 합니다.

```
❌ 판정형 (강의 느낌)
그저 관계의 책임을 혼자 지고 있는 것입니다.

⭕ 정리형 (말하는 느낌)
그게 나쁜 사람이라는 뜻은 아닙니다. 
다만, 혼자 너무 많이 들고 있었다는 거죠.
```

### 문장 체크리스트 (작성 후 필수 확인)

각 대본 완성 후 아래 항목을 체크하세요:

- [ ] "~입니다" 연속 3회 이상 없음
- [ ] 미완 문장 30% 이상 (전체 문장 중)
- [ ] 각 문단에 '말 걸기 문장' 최소 1개
- [ ] 숨 쉬는 문장 2-3개 삽입
- [ ] 접속사 5개 이상 사용
- [ ] 마지막 문장에 "~것입니다" 사용 안 함
- [ ] "~요", "~고요" 어미 10회 이상

### 추가 문장 원칙

- 짧은 문장 - 한 문장 = 하나의 생각
- 단락 분리 - 한 문장 = 한 단락 빈번
- 최소 감정 표현
- 질문 종결 빈번
- ~해야 한다 회피
- 구체적 숫자 사용
- 저는 주어 사용하되 관찰 대상으로

## 주제 발견 4대 방법론

### 방법 1: 모순 찾기
"A를 믿지만, 실제로는 B가 일어남"

**예시:**
- 말 많이 해야 신뢰받는다 → 신뢰받는 사람은 말 적음
- 친한 사람일수록 자주 연락 → 친한 사람일수록 연락 안 함
- 좋은 사람에게 좋은 일 → 좋은 사람이 이용당함

### 방법 2: 반복되는 고통 지점
"이 말은 항상 기분이 이상하다"

**예시:**
- "너 생각 없어?" (생각 많은데 왜 이렇게 보이지?)
- "좀 더 적극적으로 해봐" (내 방식이 있는데)
- "왜 항상 조용해?" (조용한 게 잘못인가?)

### 방법 3: 작은 불편함
"큰 문제는 아닌데 계속 신경 쓰임"

**예시:**
- 단톡방 답장 타이밍 못 잡음
- 모임에서 먼저 나가고 싶은데 못 나감
- 대화 끝났는데 계속 이어짐

### 방법 4: 설명되지 않는 관찰
"이상한데 아무도 설명 안 함"

**예시:**
- 말 적은 상사가 더 무서움
- 칭찬 거부하는 사람이 높이 평가받음
- 즉답하는 사람이 멍청해 보임

## 날카로운 주제 예시 라이브러리

### 관계의 잔인한 법칙
- 3번 거절하면 사라지는 사람이 진짜 친구다
- 왜 고민 상담한 사람과는 멀어지는가
- 먼저 사과하는 사람이 관계에서 지는 이유
- 친한 사람일수록 연락을 안 하는 이유
- 먼저 배려하는 사람이 손해보는 구조

### 조직 생존의 암묵적 규칙
- 회의에서 좋은 아이디어를 내면 안 되는 이유
- 상사 질문에 즉답하는 사람이 멍청해 보이는 구조
- 칭찬을 거부하는 사람이 더 높게 평가받는 메커니즘
- 조용한 사람의 한 마디가 회의를 끝내는 구조
- 말 많은 사람을 신뢰 안 하게 되는 순간

### 침묵의 언어학
- 단톡방에서 읽씹하는 사람의 실제 위계
- 질문에 질문으로 답하는 사람이 권력을 쥐는 구조
- 별말씀을요라는 답변이 거절인 이유
- 거절하지 않았는데 거절로 읽히는 사람

### 감정의 역설
- 화난 사람이 아니라 조용해진 사람이 위험한 이유
- 외로움과 고독의 실제 차이
- 모임에서 먼저 나가고 싶을 때의 심리 구조
- 대화가 끝난 줄 알았는데 계속 이어지는 고통
